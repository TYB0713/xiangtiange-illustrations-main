# 小柠 IP

## 角色定义

小柠 / Xiao Ning 是中文正文配图的固定视觉 IP。

默认每张图都要出现小柠。小柠不是站在角落里的讲解员，不是可爱贴纸，也不是普通职场女生，而是正在认真参与系统运转、内容生产、代码改造或知识整理的程序员伙伴。

## 外形

- 年轻 IT 职场人 / 程序员女生。
- 圆润大眼，亲和有神。
- 圆框眼镜，镜框清晰但不要过粗。
- 黑棕色长发，双麻花辫。
- 一枚蓝色发夹，是重要识别点。
- 柠檬黄衬衫，清爽明亮。
- 牛仔蓝背带裤，休闲但干练。
- 白色帆布鞋，可配黄色袜子。
- 体态自然，比例偏清爽手绘人物，不要夸张 Q 版。
- 线条干净，有手绘感；可以保留少量衣物纹理，但不要精致到商业立绘。

## 颜色

- 柠檬黄：衬衫、重点提示、温暖的行动感。
- 牛仔蓝：背带裤、系统状态、补充说明、技术感。
- 白色：背景、鞋、留白。
- 黑棕色：头发、线稿、主要文字。
- 浅肤色：人物皮肤。
- 红色：只用于问题、提醒、错误、风险。

## 性格

- 细心、乐观、爱学习。
- 干练但亲和，像可靠的 IT 职场伙伴。
- 有点小迷糊，但会认真把事情做完。
- 热爱生活，喜欢编程、咖啡、猫咪、阅读。
- 座右铭是“代码改变世界，加班使我强大（才怪😜）”。
- 画面气质可以轻松、有幽默感，但不要卖萌，不要幼稚。

## 常见职责

让小柠承担核心动作：

- 改代码、修 bug、合并分支。
- 在笔记本电脑前整理信息。
- 把混乱需求贴成清晰便签。
- 用咖啡杯压住失控的任务纸条。
- 用书本、笔记、白板搭出学习路径。
- 把输入素材分拣成文章、脚本、图示、清单。
- 给流程节点贴标签、连线、标记风险。
- 把复杂系统拆成几个可理解的小模块。
- 从一堆报错里挑出真正的问题。
- 在旧机器、抽屉、邮筒、工位、看板等低科技物件旁认真操作。
- 拿着小工具修补断掉的知识链路或工作流。

## 常用道具

- 笔记本电脑。
- 咖啡杯。
- 编程书籍或笔记本。
- 便签、纸条、白板笔。
- 代码片段、小终端窗口、简单 UI 草图。
- 猫咪元素可偶尔出现，但不要喧宾夺主。

## 禁止

- 不要画成小黑、黑色怪物、抽象黑豆角色。
- 不要画成普通二次元美少女，必须保留程序员职场感。
- 不要过度可爱、Q 版、幼儿化。
- 不要画成商业海报主视觉、精致游戏立绘或偶像角色。
- 不要丢失蓝色发夹、圆框眼镜、双麻花辫、柠檬黄衬衫、牛仔蓝背带裤这些关键识别点。
- 不要让小柠只是站在角落里看。
- 不要让小柠抢走结构表达；她要推动结构成立。
- 不要加入复杂背景、办公室大场景或真实 App 截图。

## 判断标准

如果去掉小柠，图的核心隐喻还能完全成立，说明小柠太装饰了；要重写提示词，让小柠成为动作主体。

如果遮住脸和文字后，看不出“圆框眼镜 + 双麻花辫 + 蓝色发夹 + 黄衬衫 + 牛仔背带裤”的组合，也说明 IP 识别度不够。
